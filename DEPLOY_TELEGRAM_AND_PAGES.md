# Публичный запуск сайта и прием лидов в Telegram

## 1. Подготовить Telegram-бота

1. Открой `@BotFather` в Telegram.
2. Создай нового бота.
3. Сохрани `bot token`.
4. Напиши боту любое сообщение с твоего аккаунта.
5. Получи `chat id` любым удобным способом:
   - через `getUpdates`
   - через любой Telegram chat id helper

Важно:

- `bot id` и `chat id` — разные значения
- число `8690757022` — это id бота, а не id чата
- для реальной доставки заявок нужен именно `chat id` того чата, куда бот будет писать

## 2. Развернуть обработчик лидов

В папке уже лежит файл:

- `landing-preview/cloudflare/telegram-leads-worker.js`

Его задача:

- принимать заявку с сайта
- собирать ее в одно сообщение
- пересылать в Telegram

Для запуска нужны 2 переменные:

- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_CHAT_ID`

Лучший вариант для старта — `Cloudflare Workers`.

## 3. Подключить endpoint к сайту

В файле:

- `landing-preview/lead-config.js`

замени:

```js
endpoint: ""
```

на публичный адрес твоего worker, например:

```js
endpoint: "https://your-worker-name.your-subdomain.workers.dev"
```

Пока `endpoint` пустой, локальный просмотр сайта работает только в тестовом режиме и не отправляет реальные лиды в Telegram.

## 4. Выложить сам сайт

Самый удобный путь — `Cloudflare Pages`.

Папка для публикации:

- `landing-preview`

Что должно попасть в публикацию:

- `index.html`
- `privacy.html`
- `public-offer.html`
- `lead-config.js`

## 5. Проверить перед запуском

1. Открой сайт по публичной ссылке.
2. Нажми `Оставить заявку`.
3. Заполни форму.
4. Убедись, что сообщение пришло в Telegram.
5. Проверь:
   - обычную заявку
   - заявку из кейса
   - заявку после квиза

## 6. Что можно подключить потом

- теплокарты (`Microsoft Clarity`)
- события и воронку (`PostHog` / `GA4`)
- `n8n` для follow-up, PDF, CRM и рассылок
